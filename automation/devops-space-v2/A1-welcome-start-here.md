# 👋 Welcome — Start Here

This space is the single instruction manual for how our team plans, builds, tests, documents, and releases. You never need a meeting or a DM to find out "how do we do X here" — if it's not in this space, it isn't our process yet (and *that* is a gap you should report — see Health Check).

## 1. The 60-second version

Think of our delivery system as an airport ✈️. **Azure Boards** is air-traffic control (what flies, in what order). **GitHub** is the runway (where things actually take off). **This space** is the operations manual every pilot and crew member follows. Nothing takes off without a flight plan (a work item), nobody freelances on the runway (no untracked work), and every landing is logged (evidence on the item).

## 3. The five folders (what lives where)

🚀 **Start Here** — orientation (you are here). 🔁 **How We Work** — the daily/sprint processes. 📐 **Standards & Reference** — the rules we build against. 🗂 **Records & Live Data** — links, logs, plans, live status. 👥 **Team Hub** — people, roles, forums.

One question = one folder = one page. If you ever find the same answer written in two places, report it on Health Check — duplicates are how documentation rots.

## 5. The three rules that make everything else work

1. **No untracked work** — every task is a work item before anyone touches a keyboard.
2. **No chat-only decisions** — decisions live in the Decisions database, same day.
3. **Evidence travels with the work** — every state change carries proof (PR, test run, demo). You will show your work at Show & Tell — that's not surveillance, it's how small contributions get seen and credited.

## 6. Where the tools are

Every URL (board, repos, pipelines, Kibana, Vault, dashboards, on-call) lives in ONE place: the **Link Registry**. Bookmark it. Pages in this space never carry their own copies of links — exactly like a building has one fuse box instead of wires hanging from every wall.

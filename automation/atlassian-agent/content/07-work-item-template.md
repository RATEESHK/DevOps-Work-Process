> This is the standard template for every refined DevOps work item. Copy the block below into the work-item description in Azure Boards (or save it as a Confluence template via **Space settings → Content tools → Templates**).

```
Title:
[Short, actionable summary]

Problem / Objective:
[What problem are we solving, or what are we establishing?]

Background / Context:
[Relevant links, repository names, pipeline names, environment details, or assessment findings]

Scope:
[What is in scope for this item?]

Out of Scope:
[What is explicitly not included?]

Value / Outcome:
[Why does this matter? What improvement will it create?]

Dependencies:
[Known dependencies, teams, approvals, access, or decisions needed]

Acceptance Criteria:
- [Criterion 1]
- [Criterion 2]
- [Criterion 3]

Definition of Ready Check:
- [ ] Objective is clear
- [ ] Scope is understood
- [ ] Owner identified
- [ ] Dependencies captured
- [ ] Acceptance criteria defined
- [ ] Estimate added
- [ ] Supporting context attached

Notes / Links:
[Confluence links, repo links, assessment references, related work items]
```

## How to use it

1. Paste into the work item description.
2. Replace every `[...]` placeholder.
3. Tick the Definition of Ready boxes before bringing the item to sprint planning.
4. For technical items, also follow the **S.C.A.L.E.D.** format in the description so reviewers get scale, compliance, action, leverage, effect, and diffusion at a glance.
